# DOOM Reward Redesign (Plan C)

最終確認: 2026-03-10

## 概要

DOOM 報酬システムを、現行の「キル即時支給 + 被弾でストリークリセット」から、
**1マップ = 1ラウンド** の未確定プール方式へ再設計する。

この案の目的は次の 3 点です。

- 経済バランスを壊しにくくする
- 高報酬の緊張感を維持する
- 保存状態の増加を避け、実装を過度に複雑化しない

本案では Elin 本体キャラクターへのデバフや呪いは導入しない。
最悪の損失は **casino_coin の喪失** に限定する。

## 現行仕様の問題

現行の報酬は以下の性質を持つ。

- キル時に即座に casino_coin を付与する
- 1 キル目は 100 チップ
- 無傷キル継続で 2 倍ずつ上昇し、1 キル最大 5000
- 被弾でストリークが 0 に戻る
- ステージクリアで +5000
- ボスマップクリアで +10000
- ESC 退出でも、それまでに得たチップはそのまま保持される

この構造には次の問題がある。

- 上手いプレイヤーが短時間で大量のチップを確定取得しやすい
- 「欲張るか、切り上げるか」の判断が弱い
- 途中退出が安全で、リスクの圧が弱い
- 報酬がその場で確定するため、カジノ的な不安定さが薄い

## 採用方針

採用するのは **案C**。

### 案Cの定義

- **1マップ = 1ラウンド**
- ラウンド開始時にレートを選ぶ
- キル報酬は即時支給せず、そのマップ専用の **未確定プール** に貯める
- 被弾時にプールの一部を失い、倍率を落とす
- マップクリア時にだけプールを精算して casino_coin を付与する
- 死亡または ESC 退出時は、そのマップの未確定プールを失う
- 次マップにプールは持ち越さない
- save / continue に、レート・プール・倍率状態は保存しない

### 案Aを採用しない理由

案Aは「1セッション全体 = 1ラウンド」であり、プールやレートの持ち越し管理が必要になる。
その場合、次が必要になる。

- 未確定プールの save 追加
- レート状態の save 追加
- CONTINUE 時の復元仕様
- ESC 退出時の扱いの厳密化

これは保存仕様と整合仕様を増やし、今回の目的である「面白さを保ったままシンプルにする」から外れる。

## ルール仕様

### 1. ラウンド単位

- ラウンドはマップ開始時に始まる
- ラウンドは次のいずれかで終わる
  - ステージクリア
  - 死亡
  - ESC 退出

ラウンド終了時の扱い:

- ステージクリア: 未確定プールを精算して casino_coin を付与
- 死亡: 未確定プールは全損
- ESC 退出: 未確定プールは全損

### 2. レート

レートは **ラウンド開始時にのみ選択** する。
ラウンド中は変化しない。
次マップでは再度選び直す。

初版は 3 段階固定とする。

- LOW
- MID
- HIGH

レートによって変える値:

- 参加コスト
- 1 キルあたりの基本プール加算量
- 被弾時のプール減衰率

レートは save に保存しない。
CONTINUE 後は、その時点のマップに対して新しいラウンドとして再選択する。

### 3. 参加コスト

ラウンド開始時、選択したレートに応じて casino_coin を即時徴収する。

- 所持チップ不足のレートは選択不可
- 参加コストは返却しない
- 死亡 / ESC / クリアのいずれでも、参加コスト自体は確定消費

この設計により、負けたときの最低損失が明確になる。

### 4. 未確定プール

敵撃破時の報酬は即時支給せず、現在ラウンドのプールに加算する。

プールの特徴:

- 現在マップ内でのみ有効
- クリアまで現実の所持チップにならない
- 死亡または ESC 退出で消える

### 基本加算

初版はレートごとの固定値でよい。
例:

- LOW: 25
- MID: 60
- HIGH: 120

この値に倍率を掛けて加算する。

### 5. 倍率

レートとは別に、戦闘中だけ動く **倍率** を持つ。

倍率は無傷キルで上がり、被弾で落ちる。
指数倍ではなく固定段階制にする。

推奨段階:

- x1.0
- x1.5
- x2.0
- x3.0
- x4.0

仕様:

- キル時、被弾していない状態なら 1 段上がる
- 上限は x4.0
- 被弾で x1.0 に戻る

初版では「被弾後しばらく倍率上昇停止」は入れない。
これは面白いが、状態管理を増やすため第2段階とする。

### 6. 被弾ペナルティ

被弾時は、現在プールの一部を失う。

推奨:

- LOW: 20%
- MID: 27%
- HIGH: 35%

同時に:

- 倍率を x1.0 に戻す

この組み合わせにより、
「一発で全部終わらないが、無視できないほど痛い」を作る。

### 7. ステージクリア精算

マップをクリアした時だけ、そのラウンドの未確定プールを精算する。

精算内容:

- 現在プールを casino_coin に変換して付与
- 現在プールを 0 に戻す
- 倍率を初期値に戻す

ボスマップの追加報酬は、現行の `+10000` を当面維持してよい。
ただし、今後の期待値次第で調整前提とする。

### 8. 死亡と ESC 退出

### 死亡

- 現在プールは全損
- 参加コストは戻らない
- ラウンド終了

### ESC 退出

- 現在プールは全損
- 参加コストは戻らない
- DOOM の進行 save は従来どおり保持
- ラウンド報酬だけ破棄して終了

ここでは「勝ち逃げ手段」としての ESC は認めない。
ESC はあくまで中断と撤退であり、未確定報酬の持ち帰り手段ではない。

### 9. CONTINUE

CONTINUE では DOOM 本体の進行だけを再開する。

再開時の扱い:

- そのマップに対して新しいラウンドを開始する
- レートを再選択する
- プールは 0 から開始
- 倍率は初期値から開始

つまり、報酬状態は save に一切保存しない。

これにより、追加保存項目は不要になる。

## UI 方針

### 1. 事前選択

DOOM 起動時または CONTINUE 時に、レート選択を挟む。

- LOW
- MID
- HIGH

表示内容は簡潔でよい。

- 参加コスト
- 基本報酬
- 被弾時ロスト率

### 2. プレイ中 HUD

表示は最小限に絞るが、**現在の損益が直感で分かること**を最優先にする。

常時表示の主軸は `RUN` とする。

- `RUN`
- `MULTI`
- `RATE`

`RUN` は、そのラウンドで最終的に持ち帰れる見込み額を表す。
参加コスト込みで計算するため、ベット直後はマイナスから始まる。

例:

- `RUN -500`
- `RUN +1240`
- `MULTI x2.0`
- `RATE MID`

内部実装では `POOL` を持っていてよいが、プレイヤーへの常時表示は `RUN` を優先する。
これにより「今このラウンドはプラスかマイナスか」が一目で分かる。

### 3. フィードバック

イベント時は、**加算/減算の実数値と理由**を明示する。

キル時:

- `+90 KILL`
- `+180 KILL x2.0`

被弾時:

- `-342 HIT LOSS (27%)`
- `MULTI RESET`

クリア時:

- `CASH OUT +xxxx`
- 付与音 / ポップアップ

死亡 / ESC退出時:

- `POOL LOST`

必要なら `POOL LOST 2180` のようにロスト実数値も出してよい。

### 4. レート選択画面の表示

レート選択では、そのラウンドの条件を行ごとに明示する。

例:

- `LOW   ENTRY 500   BASE 25   HIT LOSS 20%`
- `MID   ENTRY 2000  BASE 60   HIT LOSS 27%`
- `HIGH  ENTRY 5000  BASE 90   HIT LOSS 35%`

選択中の補助説明として、次のような一文を出す。

- `RUN starts at -2000. Clear the map to cash out.`

## 実装方針

### 1. DoomBackend の責務縮小

backend は報酬総額を持たず、戦闘イベントと統計だけを持つ。

維持:

- total kills
- max kill streak
- current kill streak
- clear event count
- boss clear event count
- kill event queue
- 被弾検出

削除 / 廃止対象:

- `KillChipPayout`
- `GetKillRewardByStreak()` 前提の即時支給
- キルごとのチップ確定ロジック

### 2. DoomSessionManager に報酬状態を集約

session manager が次を持つ。

- 現在レート
- 現在プール
- 現在倍率段階
- 処理済み kill 数
- 現在ラウンドの確定報酬合計

また、次を担当する。

- 参加コスト徴収
- キル時プール加算
- 被弾時プール減衰
- クリア時精算
- 死亡 / ESC 時プール破棄
- HUD 更新
- ログ文言更新

### 3. Save 変更を避ける

`DoomPersistentSaveStore` は原則そのままとする。

- 報酬状態は保存しない
- save summary の `TotalChips` は確定済み報酬だけを反映するか、別途見直す

この案では報酬状態を save に乗せないことが重要な簡略化ポイントである。

### 4. 現行との互換

旧 save との特別な移行は不要。
理由:

- 報酬状態を保存しないため
- 旧 save を continue しても、新ルールでそのマップの新ラウンドを始めればよいから

## チューニング方針

初版では以下を優先する。

- ルールが分かりやすいこと
- クリア時だけ確定すること
- 被弾がちゃんと怖いこと
- 保存状態を増やさないこと

後から足せるもの:

- 被弾後の倍率上昇停止
- ボーナス部屋 / 専用換金地点
- 高レートの追加
- マップ種別ごとの報酬係数

## 非採用項目

今回の初版では採用しない。

- Elin 本体キャラへのデバフ
- 呪い
- 借金取りイベント
- 負債の永続化
- セッション全体持ち越しプール
- 報酬状態の save / continue 復元
- ESC での換金

## 期待される効果

- 即時確定報酬を減らし、経済バランスを崩しにくくする
- 被弾と撤退に明確な痛みを与える
- 1マップ単位で緊張感を作る
- 保存仕様を増やさずに済む
- 実装の主戦場を `DoomSessionManager` に寄せられる
