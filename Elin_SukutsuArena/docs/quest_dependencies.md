# クエスト依存関係（最新）
このドキュメントは「現在の依存関係の読み方」と「更新手順」を示します。
個別の依存グラフは自動生成に任せ、ここでは運用ルールのみ記載します。

## 依存関係の評価ルール
1. **未完了**であること。
2. **フェーズ条件**: `quest.Phase <= current_phase`。
3. **必須クエスト**がすべて完了していること。
4. **必須フラグ条件**を満たすこと。
5. **ブロック対象**のクエストが未完了であること。

## 依存の正を置く場所
- 依存関係の正は **クエスト定義**に置く。
- フラグを増やす前に「クエスト完了から導出できないか」を検討する。

## 更新手順
1. `tools/arena/data/quests.py` を更新。
2. ビルドして `Package/quest_definitions.json` を生成。
3. 必要なら `docs/quest_dependencies.md` を更新。

## 第二部以降のフラグ設計（要点）
- フラグは最小化し、導出可能な状態は保存しない。
- フェーズで表現できる分岐はフラグ追加せず、フェーズ駆動で吸収する。
- 一時的な状態はセッションフラグで管理し、永続化しない。
- 二択以上の状態は enum を優先し、bool 乱用を避ける。
- 追加前に「既存フラグやクエスト完了から導出できないか」を必ず確認。

## 型/APIバリデーション（ゲーム更新対策）
- `GameDependency` を基に、ゲーム側アセンブリの型・メンバー存在を検証する。
- HarmonyPatch は `new[]{ typeof(...) }` から署名を取得し、引数型まで一致検証。
- 実行: `cd tools && uv run python builder/verify_game_api.py --ci`
- 必須: .NET 8 Runtime / `ELIN_GAME_DIR` または `ELIN_MANAGED_DIR` の設定
- CWL 未導入環境では `Cwl.*` は INFO 扱い（導入環境で検証）。
